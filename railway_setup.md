# Настройка Railway для сохранения данных

При деплое на Railway данные из директории `data/` могут теряться при перезапуске контейнера. Для сохранения данных между деплоями рекомендуется использовать Railway Volumes.

## Способ 1: Railway Volumes (Рекомендуется)

1. В панели Railway откройте ваш проект
2. Перейдите в раздел **Volumes**
3. Нажмите **+ New Volume**
4. Укажите:
   - **Mount Path**: `/app/data`
   - **Name**: `bot-data` (или любое другое имя)
5. Сохраните изменения

Теперь данные в директории `data/` будут сохраняться между деплоями.

## Способ 2: Переменные окружения для критичных данных

Если вы не хотите использовать Volumes, можно хранить критичные данные в переменных окружения:

1. В панели Railway откройте ваш проект
2. Перейдите в раздел **Variables**
3. Добавьте переменные окружения для:
   - `BOT_TOKEN` - токен бота
   - `ADMIN_IDS` - ID администраторов (через запятую)

## Важно

- Файл `default_schedule.txt` будет создан автоматически при первом запуске
- Файл `admins.txt` будет создан автоматически с администраторами из `config.py`
- Остальные файлы (`employees.txt`, расписания и т.д.) будут создаваться по мере использования бота

## Резервное копирование

Рекомендуется периодически делать резервные копии директории `data/`:
- Скачивайте файлы через Railway CLI или веб-интерфейс
- Или настройте автоматическое резервное копирование в облачное хранилище
